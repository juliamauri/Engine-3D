What should you expect next for RedEye?

The end goal is to be able to produce graphic applications as RedEye is a video game engine, but it is still far from being finished. We intend to continue working on adding and improving features we deem interesting before working on the final deployment procedures.

1. Expand Physics

The physics module only handles particle simulations for now although it was built with the intent to operate with colliders and rigid bodies inside the scene.

2. Rendering APIs

RedEye currently uses OpenGL to communicate with the GPU. We now understand how most of it operates, but we are curious about how the other mainstream frameworks differ. We plan on abstracting our rendering module and allow support for DirectX and Vulkan technologies and explore their inner workings.

3. Advanced Graphics

Since adding deferred shading pipeline, we have massively improved scene lighting. We still want to add to the system and set up procedures for ray tracing, PBR support, and other post-processing effects such as ambient occlusion, shadows and bloom effects.

4. Multi-threading

Everything runs on the same thread. We use the remaining milliseconds between frames to iterate audio and other asset file changes. This simple approach can be optimized through the use of the CPU’s threads. It can also be applied to the particle system for parallel iteration.

5. Networking

We previously have worked on simple applications communicating with each other, but we are curious as to how we can adapt our socket communication patterns into engine development for client-server connections.

6. Cross Platform

Video games come across many platforms. We would like to explore how different architectures differ between them and how we could manage code throughout operative systems. We currently use Electronic Arts’ standard containers to abstract Windows’ platform dependencies (and improve debug mode performance) but some other dependencies still remain. Most libraries used are selected due to their cross platform nature.

7. Scripting

Once we are comfortable with the engine’s capabilities, the final marker is set by the scripting implementation we decide on. We have worked with simple C# solutions before, but Unreal Engine’s visual scripting using nodes feels far more challenging and fun approach.
