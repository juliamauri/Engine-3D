Welcome to RedEye Engine!

RedEye is an open source C++ 3D Game Engine. We are working on adding new interesting features before setting the game deployment pipeline. Feel free to try out our latest released version or dig into our source code repository and explore the engine’s structure.

New in v5.0

RedEye’s new Particle System is out!

In the latest version of RedEye, we implemented a particle pipeline. Open the particle workspace to customize how particles spawn, collide, and render. You can bind particles to a fixed shape, apply different opacities using a curve and even give them lights to brighten your scenes. Check the latest release notes here.

Physics

Particles can activate their colliders to check collisions against boundaries or against each other. Physics module controls simulation update timings and offers different configurations for free, fixed or timed-step updates.

Rendering

Customize how your simulations are drawn. Change their color and opacity using curves or base them on particle properties such as distance traveled, life time and speed. GPU memory usage has been optimized to render up to 508 light particles through deferred shading or 203 without deferring.

Resource Management

Emission assets are split into emitter and rendering properties. Combine different configurations to reduce memory usage. Different simulations sharing either configuration will update both on edit for simpler scene iteration.

Primitives

Particles can render primitive shapes or other dropped geometry. For improved primitive use, we set up a manager that handles processed primitive data as another RedEye resource. This fastened VAO allocation and referencing procedures.

Editor

Easily customize particles with a new window in the editor. Particle Workspace counts with an independent viewport to preview in real time how changing properties affect the simulation. Next to the viewport are the spawning and rendering fields to modify them together with independent playback controls.

Profiling

Particles being highly data-intensive require extra care as optimizations may have exponential effects. To measure performance efficiently, ReyEye counts with different binary solutions that measure the engine's performance and outputs results to a json file. We use Tableau to read, visualize and compare this data.
